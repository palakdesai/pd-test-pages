Step-by-step guide

Build & publish a site
with Claude + GitHub Pages

Prompt Claude to generate HTML, push it to a GitHub repo, then point a custom subdomain at it with a CNAME record.

1
Claude

"Create a single-page HTML slideshow website about the Taj Mahal with exactly 5 slides. Use a split-screen layout on each slide: the left half is a full-height photo panel with a real building photograph, and the right half is the content panel with text and facts."

Claude outputs a single self-contained index.html file — no external build tools needed. Copy the entire response and save it as a file.
2
GitHub Pages

Save the file to your GitHub repository

# Repo: github.com/palakdesai/pd-test-pages # 1. Save Claude's output as index.html in repo root # 2. Commit and push to main branch # 3. Go to: Settings → Pages # Source: Deploy from branch → main → / (root) # 4. Click Save
📁 pd-test-pages ⚙️ Settings → Pages index.html in root
GitHub Pages will publish your site at palakdesai.github.io/pd-test-pages within 1–2 minutes of enabling it. You'll see a green tick in the Pages settings when it's live.
3
DNS · CNAME

Add a CNAME record to point your subdomain at GitHub

Type
CNAME
Name / Host
taj
Value / Points to
palakdesai.github.io
TTL
30 min
Two more things to complete the connection:

1. Add a file named CNAME (no extension) to the root of your repo. Its only content should be your custom domain, e.g. taj.yourdomain.com

2. In GitHub → Settings → Pages → Custom domain, enter that same domain and click Save. GitHub will verify DNS and enable HTTPS automatically.
Live at taj.yourdomain.com DNS propagation: up to 30 min
